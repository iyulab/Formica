# Formology 양식학의 이론적 기반

## 개요

이 문서는 Formology를 **양식학(Formology)**으로 학문적으로 정립하는 8대 핵심 명제를 제시합니다. 철학적 논의보다는 **실천 가능한 원칙**에 초점을 맞춥니다.

> **목적**: Formology의 이론적 토대를 명확히 하되, 현업이 이해하고 적용할 수 있게

---

## Formology의 8대 핵심 명제

### 명제 1: 목적적 본질론

**핵심**: 양식의 본질은 목적으로 정의된다

```
양식의 정체성 = 그것이 달성하는 목적

If 목적(F₁) ≠ 목적(F₂), then F₁ ≠ F₂
```

**예시**:
- 주문서 vs 견적서: 목적이 다름 → 다른 양식
- 주문서: 구매 의사 확정
- 견적서: 가격 정보 제공

**실천**:
- 양식 설계 시 목적 먼저 정의
- 목적 없는 필드는 제거
- 목적 변경 시 양식 재검토

---

### 명제 2: 요소 삼분론

**핵심**: 양식 요소는 본질적/부가적/설명적으로 구분

```
모든 요소 e는 다음 중 하나:
- 본질적 (없으면 목적 달성 불가)
- 부가적 (있으면 좋음)
- 설명적 (안내, 도움말)
```

**판별 기준**:
```
본질적: "이것 없이 여전히 [양식]인가?" → No
부가적: "이것이 있으면 더 좋은가?" → Yes
설명적: "이것은 입력하는 것인가?" → No
```

**실천**:
- 모든 필드를 3가지로 분류
- 본질적 = 필수(Required)
- 부가적 = 선택(Optional)
- 설명적 = 안내/도움말

---

### 명제 3: 위계적 추상화

**핵심**: 서식-양식-문서는 다른 추상화 수준

**기본 2층 모델** (추천):
```
양식 (Form)          템플릿
  ↓
문서 (Document)      인스턴스
```

**확장 3층 모델** (필요 시만):
```
서식 (Format)        추상 원형
  ↓
양식 (Form)          구체 템플릿
  ↓
문서 (Document)      인스턴스
```

**실천**:
- 대부분은 2층 모델 사용
- 양식 변형이 많을 때만 3층
- 복잡성은 필요할 때만 도입

---

### 명제 4: 완비성과 적정성

**핵심**: 양식은 완비하면서 적정해야 함

**완비성 (Completeness)**:
```
완비 = 모든 본질적 요소 포함
```

**적정성 (Adequacy)**:
```
적정 = 완비 + 사용 가능 + 효율적
```

**실천**:
```
체크:
✓ 모든 본질적 요소 있는가? (완비성)
✓ 현업이 쉽게 작성 가능한가? (적정성)
✓ 필드 수가 적정한가? (효율성)
```

---

### 명제 5: 최소 충족성 원리

**핵심**: 필요한 최소한만 포함

```
이상적 양식 = 완비 + 최소

목적 달성 가능한 최소 요소만
```

**실천**:
```
원칙:
✓ 본질적 요소: 반드시 포함
✓ 부가적 요소: 비용-편익 분석 후
✓ 설명적 요소: 꼭 필요한 것만

제거:
✗ 사용하지 않는 필드
✗ "나중에 필요할지도" 필드
✗ 과도한 안내
```

**오컴의 면도날**: 필요 이상으로 복잡하게 하지 말라

---

### 명제 6: 관계의 삼중성

**핵심**: 문서 간 관계는 참조/붙임/스냅샷 3가지

| 관계 | 시점 | 동기화 | 용도 |
|------|------|--------|------|
| **참조** | 현재 | O | 실시간 정보 |
| **붙임** | 과거 | X | 증거 보존 |
| **스냅샷** | 특정 시점 | X | 시점 비교 |

**실천**:
```
선택 기준:
- 실시간 정보 필요 → 참조
- 증거 보존 필요 → 붙임
- 시점별 비교 필요 → 스냅샷
```

**예시**:
```
검사기록지:
- 제품 정보: 참조 (최신 정보)
- 검사 기준서: 붙임 (당시 기준 보존)
- 이전 검사 결과: 스냅샷 (비교용)
```

---

### 명제 7: 진화적 발견

**핵심**: 양식은 점진적으로 발견하고 개선

```
V₀: 초기 버전 (MVP)
  ↓ 사용 경험
V₁: 개선 버전 (문제 수정)
  ↓ 피드백
V₂: 성숙 버전
  ↓ 지속적 개선
V_n: 최적 버전
```

**실천**:
```
접근:
1. MVP로 시작 (완비한 최소 버전)
2. 실제 사용하며 문제 발견
3. 데이터 기반 개선
4. 점진적 진화

경계:
✗ 완벽주의 (시작 못함)
✗ 조기 최적화 (과도한 기능)
✓ 빠른 시작, 지속 개선
```

---

### 명제 8: 상호주관적 검증

**핵심**: 양식은 이해관계자 합의로 검증

**검증 프로세스**:
```
1. 초안 제시
   ↓
2. 다양한 관점 검토:
   - 현업: 실용성
   - 개발자: 구현 가능성
   - 관리자: 통제 가능성
   - 사용자: 사용성
   ↓
3. 합의 형성
   ↓
4. 양식 확정
```

**Formology의 강점**:
- 문서는 모두가 같게 봄
- 추상 개념보다 합의 쉬움
- 검증 가능한 구체적 대상

**품질 기준**:
- 이해관계자 90% 이상 합의
- 실제 업무 적용 가능
- 지속적 피드백 수용

## 실천 체크리스트

### 양식 설계 시

**1. 목적적 본질론 적용**:
- [ ] 목적이 명확한가?
- [ ] 목적이 정체성을 결정하는가?

**2. 요소 삼분론 적용**:
- [ ] 모든 요소가 본질적/부가적/설명적으로 분류되었는가?
- [ ] 본질적 요소는 필수로 표시되었는가?

**3. 위계적 추상화 적용**:
- [ ] 2층 모델로 충분한가?
- [ ] 3층 모델이 필요하면 정당화되는가?

**4. 완비성과 적정성 검증**:
- [ ] 모든 본질적 요소가 있는가?
- [ ] 현업이 쉽게 작성 가능한가?

**5. 최소 충족성 원리 적용**:
- [ ] 불필요한 요소는 제거했는가?
- [ ] 필드 수가 적정한가? (< 20개)

**6. 관계의 삼중성 적용**:
- [ ] 문서 간 관계가 명확한가?
- [ ] 적절한 관계 유형을 선택했는가?

**7. 진화적 발견 준비**:
- [ ] MVP로 시작 가능한가?
- [ ] 피드백 수집 계획이 있는가?

**8. 상호주관적 검증**:
- [ ] 현업과 함께 검토했는가?
- [ ] 합의가 이루어졌는가?

## 실전 예제: 품질검사의뢰서

### 8대 명제 적용

**명제 1: 목적적 본질론**
```
목적: 품질검사 의뢰
→ 정체성 결정됨
```

**명제 2: 요소 삼분론**
```
본질적: 제품명, LOT번호, 검사항목
부가적: 특이사항, 긴급여부
설명적: LOT번호 형식 안내
```

**명제 3: 위계적 추상화**
```
2층 모델 사용:
양식: 품질검사의뢰서 템플릿
문서: 각 의뢰 건 (QC-2024-001, ...)
```

**명제 4: 완비성과 적정성**
```
완비: ✓ 검사 의뢰 가능
적정: ✓ 3분 내 작성 가능
```

**명제 5: 최소 충족성**
```
필드 7개 (본질적) + 2개 (부가적)
= 총 9개 (적정)
```

**명제 6: 관계의 삼중성**
```
제품 정보: 참조 (실시간)
이전 검사 기준: 붙임 (증거)
```

**명제 7: 진화적 발견**
```
V1: 기본 필드만
V2: 긴급여부 추가 (사용자 요청)
V3: 첨부파일 기능 추가
```

**명제 8: 상호주관적 검증**
```
검증 완료:
✓ 현업 (품질팀): 실용적
✓ 개발자: 구현 가능
✓ 관리자: 추적 가능
```

## 자주 하는 질문

### Q1: 8대 명제를 모두 적용해야 하나?

**A**: 기본 원칙입니다. 하지만 프로젝트 상황에 따라 유연하게 적용하세요.

필수:
- 명제 1 (목적)
- 명제 2 (요소 분류)
- 명제 5 (최소 충족성)

권장:
- 나머지 명제들

### Q2: 이론이 너무 복잡한 것 아닌가?

**A**: 이론은 복잡해 보이지만 실천은 단순합니다.

```
실전 요약:
1. 목적 정의
2. 필드 분류 (본질적/부가적/설명적)
3. 필요한 최소한만
4. 현업과 함께 검증
```

### Q3: 온톨로지와 어떻게 다른가?

**A**: 출발점과 검증 방식이 다릅니다.

```
온톨로지:
추상 개념 → 모델 → 합의 어려움

Formology:
구체적 문서 → 양식 → 즉시 검증 가능
```

## 핵심 정리

### 8대 명제 한눈에

1. **목적적 본질론**: 목적이 정체성
2. **요소 삼분론**: 본질적/부가적/설명적
3. **위계적 추상화**: 2층 또는 3층 모델
4. **완비성과 적정성**: 완비하면서 적정하게
5. **최소 충족성**: 필요한 최소한만
6. **관계의 삼중성**: 참조/붙임/스냅샷
7. **진화적 발견**: MVP로 시작, 점진 개선
8. **상호주관적 검증**: 현업과 함께 확인

### 실천 핵심

```
1. 목적 먼저
2. 필드 분류
3. 최소로 시작
4. 함께 검증
5. 점진 개선
```
