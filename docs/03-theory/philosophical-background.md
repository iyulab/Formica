# Formology의 철학적 배경

## 문서 목적

이 문서는 Formology 방법론의 **철학적 토대**를 다룹니다. 실무 적용에는 필수가 아니지만, 이론적 정당성과 학문적 기반을 제공합니다.

> **독자**: 연구자, 학자, 이론적 깊이를 원하는 독자
> **실무자**: 이 문서를 건너뛰어도 Formology 적용에 문제없음

---

## 철학적 위치

### 존재론 (Ontology)

**입장**: **실재론 (Realism)**

양식과 문서는 추상적 개념이 아닌 **물리적 또는 디지털 실체**로 존재합니다.

```
플라톤적 이데아론 (온톨로지 접근):
완벽한 "Order Entity" 이념 → 현실은 불완전한 그림자

아리스토텔레스적 실재론 (Formology 접근):
구체적 "주문서" 실체 → 추상은 실체에서 유래
```

**함의**:
- 관찰 가능한 것에서 출발
- 검증 가능한 대상
- 현업과 공유 가능한 실체

### 인식론 (Epistemology)

**입장**: **경험론 (Empiricism)**

양식에 대한 지식은 경험과 관찰에서 유래합니다.

```
합리론 (Rationalism) - 온톨로지 접근:
"내가 개념화한다, 고로 모델이다"
생각 → 개념 → 추상화 → 모델

경험론 (Empiricism) - Formology 접근:
"경험이 지식의 원천"
관찰 → 문서 식별 → 양식 작성 → 모델 도출
```

**로크/흄의 경험론**:
- 모든 지식은 감각 경험에서 유래
- 선험적 개념 거부
- 검증 가능성 중시

### 방법론 (Methodology)

**입장**: **실용주의 (Pragmatism)**

이론의 진리성은 실무 적용에서의 유용성으로 판단합니다.

**William James의 실용주의**:
- "진리는 효용성으로 판단한다"
- 이론적 완벽성 < 검증 가능성
- 실천적 결과 중시

**존 듀이의 도구주의**:
- 개념은 문제 해결의 도구
- 성공적 적용이 진리 기준

## 철학적 우위의 근거

### 1. 인식론적 앵커 (Epistemic Anchor)

**문제**: 온톨로지의 관점 의존성

```
같은 "주문" 업무:

개발자 A의 모델:
Order → hasProduct → Product

개발자 B의 모델:
Customer → placeOrder → Order → containsItem → OrderItem

현업: "둘 다 이해 못하겠는데요?"
```

**Formology의 해법**: 문서는 누구나 같게 봄

```
"주문서"를 보면:
개발자 A: 주문서
개발자 B: 주문서
현업: 주문서
CEO: 주문서

→ 모두 동일하게 인지
→ 관점 의존성 없음
→ 상호주관적(Intersubjective) 실체
```

### 2. 불가지론 함정 회피

**온톨로지의 딜레마**:

```
Week 1: "Customer란 무엇인가?"
  → 구매한 사람? 잠재 구매자? 계약 당사자?
  → 합의 어려움

Week 8: 여전히 기초 개념 정의 중...
```

**철학적 분석**:
- 추상 개념의 "본질"에 대한 절대적 답이 없음
- 불가지론(Agnosticism): "진리는 알 수 없다"
- 끝없는 합의 과정 → 프로젝트 지연

**Formology의 회피**:
```
"주문서에 뭐가 들어가죠?"
→ 즉시 답변 가능
→ 현물 확인 가능
→ 철학적 논쟁 불필요
```

### 3. 간주관성 (Intersubjectivity)

**정의**: 서로 다른 주체들이 같은 대상을 공유하는 특성

**훗설(Husserl)의 현상학**:
- 의식의 지향성(Intentionality)
- 경험의 구조적 공유
- 공동 세계(Lebenswelt)

**Formology 적용**:
```
온톨로지:
"내가 보는 Entity"와 "네가 보는 Entity"
→ 다를 수 있음
→ 주관적 개념들의 협상 필요

Formology:
"내가 보는 의뢰서"와 "네가 보는 의뢰서"
→ 같음
→ 공유된 현실, 협상 불필요
```

## 아리스토텔레스 철학과의 연결

### 형상-질료론 (Hylomorphism)

**아리스토텔레스**:
- 모든 사물은 질료(Matter)와 형상(Form)의 결합
- 형상이 질료에 구조를 부여
- 형상은 목적(Telos)을 내포

**Formology 적용**:
```
양식 (Form) = 구조와 목적
문서 (Instance) = 양식이 데이터(질료)에 적용된 결과

품질검사의뢰서 (양식)
  ↓ 형상 부여
특정 검사 데이터 (질료)
  ↓ 결합
구체적 의뢰서 문서
```

### 4원인론 (Four Causes)

**질료인 (Material Cause)**:
- 문서: 데이터, 정보 내용
- 양식: 필드, 요소

**형상인 (Formal Cause)**:
- 문서: 양식의 구조
- 양식: 서식의 원형

**작용인 (Efficient Cause)**:
- 문서: 작성자, 작성 행위
- 양식: 설계자, 설계 행위

**목적인 (Final Cause)** ← 가장 중요
- 문서: 특정 업무 목적 달성
- 양식: 일반적 업무 범주 지원

### 본질과 우연

**본질적 속성 (Essential Property)**:
- 그것 없이는 그 사물이 아닌 것
- Formology: 본질적 요소 (Essential Features)

**우연적 속성 (Accidental Property)**:
- 있어도 되고 없어도 되는 것
- Formology: 부가적 요소 (Accidental Features)

**예시**:
```
"사람"의 본질: 이성적 동물
"사람"의 우연: 키, 몸무게, 머리색

"품질검사의뢰서"의 본질: 제품명, LOT번호
"품질검사의뢰서"의 우연: 특이사항, 첨부파일
```

## 현대 철학과의 대화

### 분석철학: 비트겐슈타인

**언어게임 이론**:
- 의미는 사용 맥락에서 결정
- 가족 유사성 개념

**Formology 연결**:
- 양식은 특정 업무 맥락의 "언어게임"
- 같은 업무 도메인에서 의미 공유

### 실용주의: 찰스 퍼스

**기호학 (Semiotics)**:
- 기호, 대상, 해석체
- 의미의 사회적 구성

**Formology 연결**:
```
기호 (Sign): 양식
대상 (Object): 업무 활동
해석체 (Interpretant): 문서

양식은 업무를 표현하는 기호 체계
```

### 현상학: 하이데거

**도구 존재론 (Zuhandenheit)**:
- 도구는 사용 맥락에서 의미 획득
- "손안의 존재"

**Formology 연결**:
- 양식은 업무 수행의 도구
- 사용 맥락이 양식의 의미 결정

---

## 사회구성주의적 관점

### 버거와 루크만 (Berger & Luckmann)

**사회적 실재 구성**:
1. 외재화: 인간 활동이 세계 생산
2. 객관화: 생산물이 독립적 실재로 경험됨
3. 내재화: 객관화된 세계를 재습득

**Formology 적용**:
```
1. 외재화: 업무 수행 → 문서 작성
2. 객관화: 양식이 조직의 표준이 됨
3. 내재화: 구성원이 양식을 업무 방식으로 학습
```

### 양식의 사회적 성격

**관습 (Convention)**:
- 양식은 사회적 합의의 산물
- 문화와 조직에 따라 변형

**제도화 (Institutionalization)**:
- 양식이 조직에 정착
- 새 구성원 사회화 도구

**권력 관계**:
- 누가 양식을 설계하는가?
- 양식은 조직 권력 구조 반영

## 철학적 비판에 대한 답변

### 비판 1: "과도한 단순화"

**비판**: 복잡한 업무를 양식으로 환원하는 것의 한계

**답변**:
- Formology는 모든 것을 문서로 환원하지 않음
- 문서로 표현 가능한 영역에 집중
- 암묵지, 숙련은 다른 방법 병행

### 비판 2: "형식주의 위험"

**비판**: 양식에 집착하여 본질 상실 가능성

**답변**:
- 목적 중심 접근으로 방지
- "양식을 위한 양식" 경계
- 점진적 진화로 유연성 확보

### 비판 3: "문화적 편향"

**비판**: 문서 중심은 서구/관료제 문화에 편향

**답변**:
- 다양한 문서 형태 포용 (구두, 시각 등)
- 문화적 맥락 고려한 적용
- 보편적 원리 + 문화적 실천

## 철학적 함의

### 인식론적 함의

**지식의 형태**:
```
명제적 지식 (Knowing-that): 이론, 개념
절차적 지식 (Knowing-how): 실천, 숙련
대상적 지식 (Knowing-of): 경험, 인지

Formology는 세 지식을 통합:
- 명제적: 양식의 구조 이해
- 절차적: 문서 작성 숙련
- 대상적: 실제 양식 경험
```

### 존재론적 함의

**추상과 구체의 관계**:
```
전통 철학: 추상이 구체보다 우위
Formology: 구체가 추상의 기반

추상적 개념 (Entity)
  ↑ 추상화
구체적 문서 (Document)
```

### 윤리적 함의

**투명성 (Transparency)**:
- 양식은 업무 절차를 가시화
- 권력의 투명한 행사

**책임성 (Accountability)**:
- 문서는 증거와 추적 가능성
- 행위의 책임 귀속

**공정성 (Fairness)**:
- 명시적 양식은 자의적 판단 감소
- 동일한 기준 적용

## 결론

### Formology의 철학적 정체성

**핵심 입장**:
```
존재론: 실재론 - 문서는 실체
인식론: 경험론 - 관찰에서 지식
방법론: 실용주의 - 유용성이 진리
```

### 철학적 강점

1. **검증 가능성**: 이론적 완벽성보다 실천적 검증
2. **간주관성**: 주관적 해석이 아닌 공유된 대상
3. **목적성**: 추상적 존재론이 아닌 구체적 목적
4. **진화성**: 고정된 본질이 아닌 점진적 발견

### 철학적 한계 인정

**완벽한 이론이 아님**:
- 모든 업무에 적용 불가
- 암묵지, 복잡한 추론은 보완 필요
- 문화적 맥락 고려 필요

**하지만 실용적**:
- 대부분의 업무 디지털화에 유용
- 현업과 개발자 간 소통 개선
- 빠른 적용과 검증

### 핵심 메시지

> **"완벽한 철학보다 유용한 실천"**

> **"추상보다 구체, 이론보다 검증"**

> **"문서는 우리 모두가 공유하는 현실"**
