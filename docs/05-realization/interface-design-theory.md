# 인터페이스 설계 원리

## 핵심 원칙

> **양식의 구조 = 인터페이스의 구조**

종이 양식처럼 보이게 만들면, 사용자가 즉시 이해합니다.

---

## 1. 필드 → UI 요소 사상

### 기본 매핑

| 양식 필드 | UI 요소 | 예시 |
|-----------|---------|------|
| [입력] | Text Input | 이름, 주소 |
| [숫자] | Number Input | 수량, 금액 |
| [선택 ▼] | Dropdown | 제품명, 상태 |
| [날짜] | Date Picker | 의뢰일, 완료일 |
| [체크박스] | Checkbox | 긴급, 승인 |
| [텍스트] | Textarea | 특이사항, 비고 |
| [파일] | File Upload | 첨부파일 |

### 요소 유형별 강조

**본질적 필드 (필수)**:
```
제품명* [드롭다운 ▼]
  ↑
붉은 * 표시, 명확한 레이블
```

**부가적 필드 (선택)**:
```
특이사항 [텍스트]
  ↑
밝은 색, 선택적 표시
```

**설명적 요소**:
```
LOT번호 [입력] (?)
        ↓ 클릭 시
  "형식: YY-MM-###"
```

## 2. 레이아웃 원리

### 정보 계층

```
[양식 구조]              [시각적 표현]
문서                    ← 제목 (H1, 크게)
├─ 섹션 A               ← 부제목 (H2, 중간)
│  ├─ 필드 1            ← 입력 (작게)
│  └─ 필드 2
└─ 섹션 B
   ├─ 필드 3
   └─ 필드 4
```

### 그룹화

**근접성**: 관련 필드는 가까이
```
┌──────────────┐
│ 작업자: [  ] │
│ 날짜: [  ]   │ ← 같은 그룹
└──────────────┘

[공백]

┌──────────────┐
│ 제품: [  ]   │ ← 다른 그룹
└──────────────┘
```

### 반응형

```
데스크톱:
┌─────────────────────────┐
│ 필드1 [   ]  필드2 [   ] │ ← 2열
└─────────────────────────┘

모바일:
┌─────────────┐
│ 필드1 [   ] │ ← 1열
│ 필드2 [   ] │
└─────────────┘
```

## 3. 상호작용 패턴

### 입력 보조

**자동완성**:
```
제품명: [입력...]
        ↓ 필터링
  [제품A]
  [제품AA]
  [제품AAB]
```

**의존 선택**:
```
카테고리: [전자제품 ▼]
          ↓ 동적 업데이트
제품: [노트북, 모니터...]
```

**지능형 기본값**:
```
작성자: [현재 사용자]  ← 자동
날짜: [오늘]            ← 자동
```

### 즉시 검증

```
[입력 중]
LOT번호: [ABC-123]
         ↓ 실시간
         ✓ 올바른 형식

[오류]
수량: [-5]
      ↓
      ✗ 0보다 커야 합니다
```

### 상태 표시

```
[작성] → [제출] → [승인] → [완료]
 ●────────○────────○────────○
 ↑ 현재
```

## 4. 접근성 (WCAG 준수)

### 필수 요소

**시각**:
- 색 대비 4.5:1 이상
- 텍스트 크기 조절 가능
- 아이콘 + 텍스트 병행

**키보드**:
- 모든 기능 키보드 접근
- Tab 순서 논리적
- Esc로 모달 닫기

**명확성**:
- 명확한 레이블
- 구체적 오류 메시지
- 충분한 도움말

**시맨틱 HTML**:
```html
<form>
  <label for="product">제품명*</label>
  <select id="product" required>
    ...
  </select>
</form>
```

## 5. 관계 시각화

### 참조 (Reference)

```
제품명: [제품A ▼]
        ↓ 실시간 조회
제품 정보:
  - 코드: PRD-001
  - 규격: 100x200mm
```

시각 표현: 링크 아이콘 🔗, 클릭 시 이동

### 붙임 (Attachment)

```
첨부 문서: [파일명.pdf] 📎
           [다운로드] [미리보기]

상태: 붙임 (원본과 독립)
```

시각 표현: 클립 아이콘 📎, "붙임" 배지

### 스냅샷 (Snapshot)

```
참조 시점: 2024-11-07 10:30
현재 상태: [변경됨]
           ↓
        [차이 보기]
```

시각 표현: 카메라 아이콘 📷, 타임스탬프

## 6. 오류 처리

### 명확한 메시지

**나쁜 예**:
```
✗ "입력 오류"
✗ "잘못된 값"
```

**좋은 예**:
```
✓ "LOT번호는 'YY-MM-###' 형식이어야 합니다"
✓ "수량은 1 이상이어야 합니다. 현재: -5"
```

### 위치

```
LOT번호: [ABC]
         ↓ 바로 아래
    ✗ 형식이 올바르지 않습니다
```

### 성공 피드백

```
[저장 성공]
✓ 저장되었습니다 (2초 후 사라짐)

[제출 완료]
✓ 제출 완료. 의뢰번호: QC-2024-001
```

## 7. 모바일 최적화

### 터치 친화적

- 최소 터치 타겟: 44x44px
- 버튼 간 간격: 8px 이상

### 적절한 키보드

```
숫자 필드 → 숫자 키패드
이메일 → 이메일 키보드
전화번호 → 전화 키패드
```

### 단계별 진행

```
데스크톱: 한 페이지에 모든 필드

모바일:
1단계: 기본 정보
2단계: 상세 정보
3단계: 검토 및 제출
```

## 8. 실전 예제: 품질검사의뢰서

### 레이아웃

```
┌────────────────────────────────────┐
│  품질검사의뢰서 작성               │ H1
├────────────────────────────────────┤
│  기본 정보                         │ H2
│  ┌──────────────────────────────┐ │
│  │ 의뢰번호*: [자동생성]        │ │ 읽기전용
│  │ 제품명*: [선택 ▼]           │ │ 필수
│  │ LOT번호*: [YY-MM-###]        │ │ 필수
│  │ 수량*: [0]                   │ │ 필수
│  └──────────────────────────────┘ │
│                                    │
│  추가 정보                         │ H2
│  ┌──────────────────────────────┐ │
│  │ 특이사항: [         ]        │ │ 선택
│  │ 첨부파일: [파일 선택]        │ │ 선택
│  └──────────────────────────────┘ │
│                                    │
│  [저장] [제출]                     │
└────────────────────────────────────┘
```

### 상호작용

```
1. 페이지 로드 → 의뢰번호 자동생성
2. 제품 선택 → 상세정보 표시
3. 입력 중 → 실시간 검증
4. 제출 → 최종 확인 → 완료 메시지
```

## 9. 설계 체크리스트

### 구조
- [ ] 양식 레이아웃 반영되었는가?
- [ ] 정보 계층 명확한가?
- [ ] 관련 필드 그룹화 되었는가?

### 상호작용
- [ ] 입력 방식 직관적인가?
- [ ] 피드백 즉시 제공되는가?
- [ ] 오류 메시지 친절한가?

### 접근성
- [ ] 키보드로 모든 기능 가능한가?
- [ ] 스크린리더 호환되는가?
- [ ] 색 대비 충분한가?

### 모바일
- [ ] 터치 타겟 크기 적절한가?
- [ ] 키보드 타입 올바른가?
- [ ] 단계별 진행 가능한가?

## 핵심 정리

**원칙**:
1. **양식 구조 = UI 구조**
2. **익숙함 = 사용성**
3. **접근성 = 필수**
4. **모바일 = 우선 고려**

**체크**:
- 본질적 필드: 명확히 강조
- 부가적 필드: 선택적 표시
- 설명적 요소: 필요 시 제공
- 즉시 검증, 명확한 피드백
