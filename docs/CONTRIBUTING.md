# 기여 가이드

Formology 프로젝트에 기여해주셔서 감사합니다!

## 기여 방법

### 1. 문서 개선

**오타 수정, 명확성 개선**
- Fork & Pull Request
- 변경 사항 간단히 설명

**새로운 섹션 추가**
- Issue에서 먼저 논의
- 승인 후 작성

### 2. 사례 공유

**실제 적용 사례**
```markdown
## 프로젝트 정보
- 산업: 제조/의료/물류 등
- 규모: 사용자 수, 문서 수
- 기간: 개발 기간
- 효과: 정량적/정성적 성과

## 적용 과정
- 워크숍 진행
- 도출된 문서 목록
- 주요 도전과 해결

## 교훈
- 잘된 점
- 아쉬운 점
- 다음에 개선할 점
```

### 3. 템플릿 제공

**업종별 문서 템플릿**
- 제조업, 의료, 물류, 사무 등
- 실제 사용한 문서 목록
- 양식 스케치

### 4. 도구 개발

**Formology 지원 도구**
- 워크숍 보조 도구
- 문서-ERD 자동 변환
- 코드 생성기

## Pull Request 가이드

### 제목
```
[유형] 간단한 설명

유형:
- docs: 문서 수정
- case: 사례 추가
- template: 템플릿 추가
- tool: 도구 개발
- fix: 오류 수정
```

### 설명
```markdown
## 변경 내용
- 무엇을 변경했나요?

## 변경 이유
- 왜 이 변경이 필요한가요?

## 영향도
- 어떤 문서/기능에 영향을 주나요?
```

## 커뮤니티 규칙

### ✅ 환영하는 것

- 건설적인 피드백
- 실제 경험 공유
- 질문과 토론
- 다양한 관점

### ❌ 금지하는 것

- 인신공격
- 스팸
- 상업적 광고
- 타 방법론 폄하
